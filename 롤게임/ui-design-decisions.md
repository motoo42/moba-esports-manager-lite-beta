## 3.5-A 메인 허브 프레임

- 전체 UI는 Football Manager식 메인 허브를 토대로 한다.
- 커리어 시작 이후 화면은 공통 게임 프레임을 사용한다.
- 게임 프레임은 좌측 아이콘 메뉴, 상단 헤더, 중앙 콘텐츠 영역으로 구성한다.
- 커리어 시작 화면은 예외적으로 단순 중앙 배치를 유지할 수 있다.
- 전체 톤은 밝은 웹 대시보드보다 어두운 게임 UI에 가깝게 잡는다.
- 정보 밀도는 높게 유지한다.
- 16:9 가로 화면을 1차 기준으로 구현한다.
- 기준 검증 해상도는 `1920x1080`과 `1366x768`로 둔다.
- 화면 캡처와 UI 검증은 `fullPage`가 아니라 현재 viewport 기준으로 확인한다.
- 상단 헤더, 좌측 아이콘 메뉴, 세부 메뉴, 진행 버튼은 고정 프레임처럼 유지한다.
- 스크롤이 필요한 경우 전체 페이지가 움직이는 것이 아니라 중앙 콘텐츠 영역 또는 특정 패널 내부만 스크롤한다.
- 메시지함, 로스터 목록, 일정/결과 목록처럼 목록형 정보는 주변 프레임을 고정하고 목록 영역만 위아래로 움직이는 FM식 구조를 기본으로 한다.

## 좌측 메뉴

- 좌측 메뉴는 이모티콘/로고 형태의 세로 메뉴로 표시한다.
- 메뉴 항목은 다음과 같다.
  - 메시지함
  - 로스터 관리
  - 훈련
  - 스카우트
  - 대회
  - 랭킹
  - 기타
- 선택된 메뉴는 테두리와 배경으로 강조한다.
- 선택된 메뉴에 따라 세부 메뉴 영역이 바뀌는 구조로 설계한다.
- 3.5-A에서는 세부 메뉴 자리와 기본 항목만 구현한다.

## 상단 헤더

- 상단 헤더에는 팀명, 현재 시즌/주차/활성 대회, 진행 버튼을 표시한다.
- 진행 버튼 위치는 항상 우측 상단으로 고정한다.
- 버튼 이름은 상태에 따라 바뀐다.
  - 평시: `진행`
  - 경기 주간/경기 직전: `플레이`
  - 경기 직후: `계속`
- 3.5-A에서는 버튼 위치와 표시만 구현하고, 실제 날짜 진행 로직은 4번에서 연결한다.

## 메인 허브

- 메인 허브는 위쪽 `현재 로스터` 패널과 아래쪽 `매치 허브` 패널로 구성한다.
- `현재 로스터` 패널은 선발 5인만 표시한다.
- 포지션 표기는 `TOP / JGL / MID / BOT / SUP`를 사용한다.
- 각 포지션은 카드형으로 표시한다.
  - 포지션명
  - 선수 이미지 또는 임시 플레이스홀더
  - 선수 닉네임
- 폼/피로도/사기는 3.5-A에서는 표시하지 않는다.

## 매치 허브 패널

- 아래쪽 매치 허브 패널은 날짜 상태에 따라 다른 내용을 보여준다.
  - 경기 직전: 매치 프리뷰
  - 경기 직후: 매치 리뷰
  - 둘 다 아니면: 다음 경기
- 3.5-A에서는 평시/프리뷰/리뷰 패널 자리를 만들고, 실제 상태 전환은 4번에서 구현한다.
- 상대 분석은 별도 스카우트 화면보다 메인 허브 프리뷰 안에 먼저 표시한다.
- 상대 스타일은 우리 팀 전략과 같은 6개 스타일 집합을 사용한다.
  - 공격 지향
  - 템포 지향
  - 운영 지향
  - 시야 중심형
  - 후반 설계형
  - 균형 전술
- 스타일 간 상성은 내부 점수로 계산하되, UI에는 숫자 대신 우호/중립/불리와 유리한 스타일/불리한 스타일 목록으로 표시한다.
- 경기 전 전망은 정확한 승률 숫자 대신 등급으로 표시한다.
  - 강한 우세
  - 우세
  - 백중세
  - 열세
  - 강한 열세
- 핵심 라인은 상대 로스터에서 폼이 가장 높은 선수의 포지션으로 표시한다.
- 상대 로스터 데이터가 아직 없는 팀은 상대 스타일별 기본 핵심 라인을 임시 표시한다.
- 프리뷰 전력과 경기 시뮬레이션에는 현재 선발 5인의 폼, 피로도, 사기 상태를 실제 반영한다.

## 대회 현황 화면

- 화면 제목은 범용 명칭이 아니라 현재 대회명을 직접 사용한다.
- LCK Cup 화면은 상단 요약, 전체 순위표, Baron/Elder 그룹 현황, 일정/결과, 토너먼트 브래킷으로 구성한다.
- 전체 순위표 컬럼은 `순위 / 팀명 / 경기 / 세트득실`로 간결하게 유지한다.
- 그룹 현황은 항상 표시한다.
- 토너먼트 브래킷은 아래에서 위로 올라가는 구조가 아니라 왼쪽에서 오른쪽으로 진행되는 구조를 사용한다.
- 브래킷 패널은 16:9 와이드 패널로 만든다.

## 로스터 관리 화면

- 시즌 중 로스터 화면은 상단 선발 라인업과 하단 후보 목록을 분리해 표시한다.
- 선발 5인 교체는 로스터 화면에서 선수 카드를 드래그 앤 드롭하는 방식을 우선 목표로 한다.
- 드래그 앤 드롭은 후보 선수 카드를 선발 슬롯으로 옮기는 구조를 기본으로 한다.
- 드래그 앤 드롭 1차 구현은 같은 포지션 선수만 해당 포지션 선발 슬롯으로 이동할 수 있게 한다.
- 선발 슬롯끼리 직접 맞바꾸는 방식은 1차 구현에서는 허용하지 않는다.
- 후보 목록은 선발 라인업에 포함되지 않은 계약 선수만 전부 펼쳐두되 `TOP / JGL / MID / BOT / SUP` 순서로 나열한다.
- 선수 기본 카드에는 닉네임, 포지션, 오버롤을 표시한다.
- `선발/후보` 배지는 카드 안에 표시하지 않는다. 상단 선발 슬롯과 하단 후보 목록의 영역 구분으로 역할을 전달한다.
- 선수 기본 카드에는 폼, 피로도, 사기 지표를 함께 표시한다.
- 별도 `상세` 또는 `(i)` 버튼은 선발/후보 배지와 겹치기 쉬우므로 1차 구현에서는 사용하지 않는다.
- 선수 카드를 클릭하면 세부 스탯을 팝업창으로 표시하고, 우측 상단 `X` 버튼으로 닫을 수 있게 한다.
- 드래그 앤 드롭과 카드 클릭이 충돌하지 않도록 일정 거리 이상 움직였을 때만 드래그로 인식한다.
- 선발 교체는 드롭 즉시 반영한다.
- 라인업 변경은 주간 단위 운영을 전제로 한다.
- 경기 주간이라도 경기 전 준비 기간에는 선발 변경을 허용하고, 경기가 이미 진행된 리뷰 상태에서는 다음 경기부터 반영하는 방향으로 둔다.
- 교체 후 메인 허브의 선발 5인 패널과 경기 시뮬레이션 입력은 즉시 갱신된 선발 5인을 기준으로 동작해야 한다.

## 선수 상태 표시

- 사기는 숫자가 아니라 5단계 이산 상태로 관리한다.
  - `최상`
  - `중상`
  - `중`
  - `중하`
  - `최하`
- 사기 UI는 작은 원형 배지와 흰색 화살표로 표시한다.
- 화살표 방향은 단계에 따라 다르게 둔다.
  - 12시 방향: 최상
  - 1.5시 방향: 중상
  - 3시 방향: 중
  - 4.5시 방향: 중하
  - 6시 방향: 최하
- 원형 배지 배경은 사기 단계에 따라 초록색, 노란색, 빨간색 계열로 구분한다.
- 폼과 피로도는 0-100 숫자로 유지한다.
- 피로도는 낮을수록 좋고 높을수록 나쁜 값으로 해석한다.
- 경기 후 선발 선수는 승패에 따라 폼, 피로도, 사기가 즉시 변한다.
- 후보 선수는 기본적으로 사기가 계속 깎이지 않으며, 1차 구현에서는 훈련 강도에 따른 폼/피로도 변화만 받는다.
- 후보 또는 2군 선수가 선발로 콜업되면 사기가 1단계 상승한다.

## 시즌 캘린더

- 시즌 캘린더는 `로드맵`과 `달력` 두 가지 보기로 구성한다.
- 기본 보기는 `로드맵`으로 둔다.
- 로드맵은 카드형 가로 타임라인으로 구현한다.
- 로드맵 카드는 배경 이미지를 쓰지 않고, 발로란트 이스포츠 로드맵 레퍼런스의 배치감만 참고한다.
- 로드맵에는 현재 대회, 다음 대회, 완료 대회 요약을 함께 표시한다.
- 각 대회 카드는 `진행중`, `완료`, `예정` 상태 배지를 표시한다.
- 잠김 상태는 별도로 보여주지 않고 예정 상태로 흡수한다.
- 대회 카드를 클릭하면 해당 대회의 현황 화면으로 이동한다.
- 완료 대회는 우승 팀과 내 팀 결과 요약을 보여준다.
- 진행 중 대회는 현재 단계와 주차를 보여준다.
- 예정 대회는 일정 범위와 대회 성격을 보여준다.
- 아시안게임은 존재하는 시즌에만 표시한다.
- 일반 시즌 로드맵과 아시안게임 시즌 로드맵은 별도 템플릿으로 둔다.
- 달력 보기는 실제 양력 달력 구조를 사용한다.
- 2026년부터 2045년까지는 JS `Date` 계산으로 실제 요일을 반영한다.
- 경기 세부 날짜는 일정 생성 엔진이 만들며, 국내 대회는 수요일부터 일요일까지 경기일로 배치한다.
- 월요일과 화요일은 기본 준비/휴식 기간으로 둔다.
- 달력에는 우리 팀 경기만 표시한다.
- 다른 팀끼리의 경기는 대회 현황 화면의 일정/결과 패널에서 확인한다.
- AI 또는 시스템 간 경기도 실제로 생성/시뮬레이션되어 순위 경쟁에 반영되어야 한다.
- 달력의 경기 칩을 선택하면 우측 세부 패널에 해당 날짜의 우리 팀 경기 정보를 표시한다.
- 전체 프레임은 고정하고, 필요한 경우 달력/목록 패널 내부만 스크롤한다.

## 후속 UI 작업

- 로스터/계약 화면, 전략/훈련 화면, 시즌 캘린더, 순위표, 매치 프리뷰/리뷰의 세부 UI는 관련 기능 구현 시점에 점진 반영한다.
