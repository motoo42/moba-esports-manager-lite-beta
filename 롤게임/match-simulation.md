# 경기 시뮬레이션

## 1. 기본 방향

경기 결과는 팀의 종합 전력을 ELO 점수로 변환해 계산합니다.

사용자의 주요 개입 지점은 다음입니다.

- 로스터 구성.
- 전략 선택.
- 훈련 계획.
- 선수 교체.
- 상대 분석.

이 다섯 요소는 ELO 계산 전후의 보정값으로 연결합니다.

## 2. MVP 공식 초안

```text
teamPower =
  선수 평균 어빌/종합점수
  + 세부 스탯 평균
  + 케미 보너스
  + 전략 보너스
  + 랜덤 변동값
  + 이벤트 보정값

opponentPower =
  상대 팀 전력
  + 스타일 상성 보너스
  + 랜덤 변동값

teamElo = teamPower를 기반으로 계산한 팀 ELO
opponentElo = opponentPower를 기반으로 계산한 상대 ELO

winProbability = 1 / (1 + 10 ^ ((opponentElo - teamElo) / 400))
```

## 3. 결과 처리

결과:

```text
랜덤 롤 <= winProbability -> 승리
랜덤 롤 > winProbability -> 패배
```

결과는 설명 가능해야 합니다.

- "강한 운영 전략이 공격적인 상대에게 잘 통했습니다."
- "낮은 멘탈 때문에 악재 이후 경기력이 떨어졌습니다."
- "Bot 포지션 선수가 경기를 캐리했습니다."

## 4. 국제전 보정

- LPL/LEC/LTA/LCP 팀은 이벤트별로 미리 정의합니다.
- 국제전 상대는 LCK Split 상대보다 평균 전력이 높거나 스타일 상성이 뚜렷하게 설정됩니다.
- 국제전 결과 로그에는 "낯선 메타", "장거리 일정", "밴픽 적응" 같은 이벤트성 설명을 추가할 수 있습니다.

## 5. 랜덤성 미결정

아직 결정할 질문:

- 강팀이 거의 이기는 구조로 갈까?
- 약팀도 전략/상성으로 이변 가능한 구조로 갈까?
- 랜덤 변동값을 어느 정도로 둘까?
