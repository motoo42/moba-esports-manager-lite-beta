# 구현 순서 안내판

이 파일은 현재 스프린트의 방향만 빠르게 보기 위한 임시 안내판

전체 체크리스트와 완료 여부는 [docs/development-checklist.md](./docs/development-checklist.md)를 기준으로 관리
새 계정이나 새 에이전트로 이어받을 때는 [CODEX_HANDOFF.md](./CODEX_HANDOFF.md)를 먼저 확인

## 현재 위치

- 기준일: 2026-06-07
- 현재 진척도 추정: 1시즌 MVP 기준 완료, 기말 프로젝트 LCK 3시즌 작동 기준 약 70~78%, 정교한 개인 프로젝트 v1 기준 약 45~52%
- 최우선 목표: 기말 프로젝트 LCK 3시즌 작동
- 목표 완료 기준: 2026, 2027, 2028 시즌을 진행하고 2028 Worlds 종료 후 시즌 요약/스토브리그까지 안정적으로 도달해 2029 진입 직전 상태를 확인
- 최근 완료: 사용자 능력치 세부조정 1차, 일반 시즌 LCK Rounds 3-5 실제 경기 구현, 시즌 템플릿/연도별 분기, 2026 LCK 10개팀 1군+2군 선수 데이터 import
- 0번 작업 완료: 기말 프로젝트 LCK 3시즌 목표 문서화
- 1번 작업 완료: 2026 LCK 10개팀 1군+2군 전체 선수 데이터 import
- 2번 작업 완료: 2026 아시안게임 시즌과 2027/2028 일반 시즌 프로필 분기
- 3번 작업 완료: 일반 시즌 LCK Rounds 3-5 실제 경기 구현
- 3.5번 작업 완료: 사용자 능력치 세부조정 1차
- 다음 목표: 4번 2027 시즌 전체 연결

## 바로 다음 구현 순서

0. 기말 프로젝트 LCK 3시즌 목표 문서화
   - 완료
   - `CODEX_HANDOFF.md`, `docs/development-checklist.md`, `IMPLEMENTATION_ORDER.md`에 신규 최우선 목표 반영
   - 기존 장기 목표는 유지하고 기말 전 우선순위만 3시즌 작동 기준으로 재정렬

1. 2026 LCK 10개팀 1군+2군 전체 선수 데이터 import
   - 완료
   - `lck2026RosterSeeds`에 실제 명단 seed 추가
   - `lck2026RatingOverrides`에 사용자 수동 능력치 보정 구조 추가
   - 새 커리어 생성 시 `lck2026Players` 전체 풀 사용
   - 로스터 빌더 팀/포지션/1군·2군/검색 필터 추가

2. 시즌 템플릿/연도별 분기
   - 완료
   - 2026은 아시안게임 시즌
   - 2027/2028은 일반 시즌
   - 다음 시즌 생성 시 연도별 시즌 프로필 기준으로 대회 상태를 초기화
   - 일반 시즌 MSI 이후 `lck-rounds-3-5` 활성화 발판 추가

3. 일반 시즌 LCK Rounds 3-5
   - 완료
   - 일반 시즌 MSI 이후 전환
   - Rounds 1-2 기록 승계와 Legend/Rise 그룹 흐름 재사용
   - 일반 시즌 후반 라운드와 LCK Season Play-In/Playoffs, Worlds 진출권 연결

3.5. 사용자 능력치 세부조정
   - 완료
   - 3번 일반 시즌 Rounds 3-5 구현 후, 4번 2027 시즌 전체 연결 전에 진행
   - 사용자가 제공한 상위 5팀/하위 5팀 핵심 선수 메모를 기반으로 `src/data/lck2026RatingOverrides.ts` 보정
   - 2군 주요 선수와 나머지 선수는 현재 데이터 기준 밸런싱 패스로 보정

4. 2027 시즌 전체 연결
   - LCK Cup부터 Worlds, 시즌 요약, 스토브리그, 2028 전환까지 연결
   - 2026 아시안게임 시즌 상태가 2027 일반 시즌에 남지 않는지 확인

5. 스토브리그 2차 최소 고도화
   - AI 로스터 변화가 다음 시즌 팀 전력에 반영되도록 연결
   - 선수 선호도와 경쟁 입찰 최소 구현
   - 팀별 로스터 변동 로그와 요약 강화

6. 선수 성장/하락/은퇴/유망주 최소 구조
   - 시즌 종료 시 성장/하락 반영
   - 로스터 부족 방지를 위한 유망주 생성
   - 은퇴 후보 최소 로직 추가

7. 2028 시즌 반복 안정화
   - 2027 일반 시즌 흐름이 2028에도 반복되는지 확인
   - 이전 시즌 대회 상태, match id, schedule id 충돌 확인

8. 3시즌 자동 시뮬레이션 테스트
   - 2026 시작부터 2028 Worlds 종료 후 스토브리그까지 진행하는 테스트 또는 debug runner 작성
   - 실패 시 날짜/대회/단계가 드러나는 로그 제공

9. 시즌 히스토리/3시즌 요약 UI
   - 시즌별 LCK/MSI/Worlds 결과와 유저 팀 성적 요약
   - 주요 이적과 선수 변화 표시

10. 저장 안정성/마이그레이션 최소 보강
   - 로그인 사용자 ID 기반 `ownerId` 연결
   - 저장 데이터 `schemaVersion` 마이그레이션 정책
   - 3시즌 진행 후 저장/불러오기 검증

11. 발표용 UI polish와 버그픽스
   - 16:9 주요 화면 검증
   - 스토브리그, 시즌 요약, 대회 브래킷의 발표용 가독성 보강
   - 발표 중 보여줄 추천 진행 루트 정리

## 구현 기본 순서

1. 정책 결정
2. 도메인 로직 구현
3. UI 연결
4. 테스트 작성/수정
5. `npm run build`
6. 필요한 경우 `npm test`, `npm run test:system`, `npm run test:acceptance`
7. 16:9 스크린샷 저장
8. `docs/development-checklist.md` 갱신

## 화면 개발 원칙

- 16:9 가로 화면 우선
- 전체 프레임 고정
- 스크롤은 중앙 콘텐츠나 패널 내부에서만 발생
- 왼쪽 메뉴와 상단 날짜/진행 버튼은 항상 고정
- UI 변경 후 가능하면 `1920x1080` 또는 `1366x768` 스크린샷 저장
