# 아직 남은 설계 질문

다음 설계 단계에서 답해야 할 질문입니다.

## 1. MVP 범위

3주 프로젝트 MVP에서는 실제로 어디까지 구현할까?

- 1시즌 완주?
- 2-3시즌 반복?
- 20시즌 구조만 설계하고 실제 플레이는 축약?

## 2. MVP 플레이 시간

한 시즌의 MVP 플레이 시간은 어느 정도로 잡을까?

- 10-20분 압축 시즌?
- 1시간 안팎?
- 2-3시간 간소화 시즌?

## 3. LCK Split 경기 수

큰 시즌 흐름은 다음으로 확정했습니다.

```text
일반 시즌:
LCK Cup -> First Stand -> LCK Rounds 1-2 -> MSI -> LCK Rounds 3-5 -> Worlds

아시안게임 시즌:
LCK Cup -> First Stand -> LCK Rounds 1-2 -> MSI -> LCK Rounds 3-4 -> Asian Games -> Worlds
```

LCK Cup은 현실 기반 간소화 포맷으로 확정했습니다.

- 직전 시즌 순위 기반 스네이크 그룹 분배.
- 그룹 배틀 5주, 주당 5시리즈, 총 25시리즈.
- 1~4주차 Bo3, 5주차 Super Week Bo5.
- 전 경기 Fearless Draft.
- 플레이-인은 6팀 싱글 엘리미네이션, 2팀 플레이오프 진출.
- 플레이오프는 5팀 싱글 엘리미네이션.
- 결승 진출 2팀이 First Stand 진출.

아직 정해야 할 세부 질문:

- LCK Rounds 1-2의 정규시즌 경기 수를 실제 일정 그대로 넣을까, 아니면 압축할까?
- Road to MSI / 포스트시즌을 실제 브래킷 그대로 구현할까?
- LCK Rounds 3-5 또는 아시안게임 시즌의 Rounds 3-4 경기 수를 실제 일정 그대로 넣을까?
- LCK 플레이오프는 Bo5 단위로 시뮬레이션할까, 아니면 압축 경기로 처리할까?

## 4. 국제전 규모

First Stand와 MSI의 큰 구조는 다음으로 확정했습니다.

```text
LCK 상위 2팀 + LPL 상위 2팀 + 3~6위 리그 상위 1팀씩 = 총 8팀
4팀씩 2개 조
각 조 상위 2팀 진출
4강 토너먼트
```

Worlds의 큰 구조는 다음으로 확정했습니다.

```text
총 22팀
플레이인 10팀 -> 상위 4팀 조별리그 진출
조별리그 16팀, 4팀씩 4개 조
각 조 상위 2팀 8강 토너먼트 진출
8강 월드컵식 토너먼트
```

아직 정해야 할 세부 질문:

- First Stand/MSI 조별리그는 Bo1, Bo3, 단일 라운드 로빈, 더블 라운드 로빈 중 무엇으로 할까?
- First Stand/MSI 4강과 결승은 Bo5로 할까?
- Worlds 플레이인은 리그전, 스위스, GSL, 간단 점수식 중 무엇으로 할까?
- Worlds 조별리그는 Bo1/Bo3 중 무엇으로 할까?
- Worlds 토너먼트는 Bo5 전체를 게임 단위로 시뮬레이션할까?

## 5. 선수 상태값

MVP에 넣을 선수 상태값은 무엇으로 제한할까?

- 폼.
- 피로도.
- 사기.
- 부상.
- 전부 넣으면 복잡해질 수 있음.

## 6. 아시안게임 차출 효과

아시안게임 차출은 사용자 팀에 불이익을 줘야 할까?

- 피로도 증가.
- Split 3 준비 부족.
- 대신 명성/사기/시장가치 상승.

## 7. 아시안게임 대표 선발 조건

아시안게임 대표 선발 조건은 어떻게 할까?

- 포지션별 종합점수 1위.
- 최근 폼 반영.
- 팀 성적 반영.
- 나이/군 문제 반영.

## 8. 타 리그 고정 팀 변화

LPL/LEC/LTA/LCP 고정 팀 데이터는 20시즌 동안 그대로 둘까, 아니면 약간 성장/노화만 줄까?

## 9. 로스터 규모

로스터 규모는 몇 명으로 할까?

- 주전 5명만.
- 주전 5명 + 후보 1-2명.
- 포지션별 후보까지.

## 10. 훈련 주기

훈련은 어느 주기로 선택하게 할까?

- 경기 전마다.
- 이벤트 시작 전마다.
- 시즌 단위로만.

## 11. ELO 랜덤성

ELO 계산에서 랜덤성은 어느 정도로 둘까?

- 강팀이 거의 이기는 구조.
- 약팀도 전략/상성으로 이변 가능.

## 12. 커리어 저장 방식

커리어 저장은 MVP에서 무엇으로 할까?

- localStorage.
- IndexedDB.
- 서버 DB는 후순위.

## 13. 커스텀 테마 우선순위

커스텀 테마 가져오기는 MVP에 포함할까?

- 핵심 게임 루프 이후 3주차 확장.
- 최종 프로젝트 이후 개선.

## 14. 리빌딩/하위권 보상

리빌딩/하위권 보상은 어떤 형태가 좋을까?

- 유망주 우선권.
- 예산 보정.
- 신인 스카우팅 이벤트.
- 추후 구현으로 유지.
