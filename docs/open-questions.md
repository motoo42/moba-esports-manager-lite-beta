# 남은 결정사항

초기 질문 중 상당수는 이미 결정됨

이 문서는 앞으로 구현 전에 다시 확인해야 할 열린 질문만 관리

## 1. 저장 방식

현재 결정:

- localStorage 임시 저장은 현재 사용하지 않음
- Express API 서버와 MongoDB native driver를 사용
- `careerSaves` 컬렉션에 `CareerSave` 전체를 저장
- 수동 저장/새 저장/불러오기 UI는 1차 구현 완료
- 불러오기 후에는 `/hub`로 복구
- 저장 문서에는 장기 멀티유저 확장을 고려해 `mode`, `worldId`, `participants`, `ownerId`, `revision`, `schemaVersion`을 포함

남은 질문:

- 자동 저장 시점: 날짜 진행 후, 경기 종료 후, 설정 변경 후 중 어디까지 할지
- 저장 실패 시 진행을 막을지, 경고만 보여줄지
- `revision` 충돌을 어떻게 처리할지
- `schemaVersion` 마이그레이션을 언제부터 구현할지
- 로그인 도입 후 `ownerId`를 어떤 인증 ID와 연결할지
- 배포 시 dev/staging/prod DB를 어떤 방식으로 분리할지

## 2. First Stand / MSI 후속

현재 상태:

- First Stand 진행 엔진 구현 완료
- LCK Cup 우승팀/준우승팀을 First Stand LCK 대표로 연결
- 조별리그와 토너먼트 UI 실제 결과 연동 완료
- LCK Rounds 1-2 포스트시즌 우승/준우승 저장 로직 존재
- MSI 진행 엔진과 전용 대회 UI 구현 완료
- MSI 성적 기반 Worlds 보너스 시드 판정과 Worlds 20팀 참가 풀 저장 완료
- Worlds 실제 Play-In/Group Stage/Knockout 진행과 우승팀 저장 완료
- Worlds 이후 시즌 요약/재계약 브리지/다음 시즌 전환 완료

확인 필요:

- 국제대회 공통 브래킷 컴포넌트 추출 여부

## 3. 국제전 경기 단위

확정된 큰 구조:

- First Stand: 8팀, 2개 조, 조 상위 2팀, 4강/결승
- MSI: 11팀, 플레이인 4팀 싱글 엘리미네이션, 브래킷 8팀 상위조/하위조 더블 엘리미네이션
- Worlds: 20팀, Play-In 8팀, Group Stage 16팀, Knockout 8팀, 결승 우승팀 저장

남은 질문:

- LCQ placeholder 2장의 실제 출처를 어떤 방식으로 정할지

## 4. LCK 후반 시즌

현재 결정:

- 일반 시즌 LCK Rounds 3-5는 Rounds 1-2 기록을 승계하고 Legend/Rise 그룹으로 분리
- Rounds 3-5 정규 그룹은 그룹 내부 트리플 라운드로빈, 팀당 12시리즈, 총 60시리즈, BO3 Fearless
- Rounds 3-5 포스트시즌은 Rounds 3-4의 Season Play-In/Playoffs 구조를 재사용
- Worlds 진출권은 Rounds 3-5 Playoffs 최종 1~3위 기본 진출, 4위는 MSI 보너스 시드 조건부 진출로 계산

남은 질문:

- Rounds 3-5 이후 2027/2028 Worlds, 시즌 요약, 스토브리그까지 3시즌 장기 진행에서 깨지는 지점이 있는지
- 고도화 스토브리그와 선수 성장/하락을 어느 깊이까지 기말 프로젝트 범위에 포함할지

## 5. 선수 성장과 장기 커리어

남은 질문:

- 선수 성장/하락을 시즌 단위로 할지, 경기 단위로 할지
- 나이는 시즌 전환 때 1 증가하도록 구현됨. 은퇴를 언제부터 구현할지
- 군 입대 이벤트를 LCK 선수에게 언제 적용할지
- 유망주 생성 규칙
- 2군 선수 성장 보너스

남은 질문:

- 실제 챔피언 목록을 어느 범위까지 넣을지
- 패치/메타 변화를 시즌 중 몇 번 발생시킬지
- 밴픽 UI를 직접 조작 가능하게 할지
- 자동 밴픽 점수만 보여줄지
- 피어리스 시스템을 언제 실제로 적용할지
