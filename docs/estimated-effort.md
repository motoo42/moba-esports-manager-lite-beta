# 예상 소요시간

## 1. 전제

롤게임은 Codex 같은 AI 에이전트를 적극적으로 사용해 개발하는 것을 전제로 합니다.

런타임 AI는 MVP에서 사용하지 않습니다. AI는 개발 과정에서 실제 LoL 선수/팀 데이터 정리, 이벤트 카드, 밸런스 초안, 테스트 시나리오를 만드는 데 사용합니다.

MVP 전제:

- 개인 프라이빗 컨텐츠이므로 실제 LoL/LCK/Riot 명칭과 데이터 사용을 허용합니다.
- 프론트엔드 단독 구현을 우선합니다.
- 실제 선수, 팀, 이벤트, 시즌 데이터는 정적 JSON/TS 데이터로 시작합니다.
- 저장은 localStorage 또는 IndexedDB 후보로 둡니다.
- 커스텀 테마 기능은 후순위 또는 선택 기능으로 봅니다.

## 2. 전체 예상 범위

현실적인 예상:

```text
70~120시간
```

강하게 스코프를 줄인 제출형 MVP:

```text
50~80시간
```

20시즌 커리어와 밸런싱까지 꽤 만족스럽게 만들 경우:

```text
110~170시간
```

욕심낸 완성형:

```text
170~250시간
```

롤게임은 체스게임보다 체스 규칙 같은 복잡한 단일 라이브러리 연동은 덜하지만, 시스템 수가 많습니다. 로스터, 계약, 전략, 훈련, 경기 시뮬레이션, 시즌 캘린더, 국제전, 아시안게임, 장기 커리어가 서로 영향을 주기 때문에 테스트와 밸런싱 시간이 커질 수 있습니다.

## 5. 체스게임과의 차이

체스게임:

- 어려운 점은 오프닝 데이터 정확도와 체스보드 상호작용입니다.
- 핵심 규칙은 chess.js 같은 라이브러리에 많이 맡길 수 있습니다.
- 데이터 검수는 까다롭지만 기능 루프는 비교적 선명합니다.

롤게임:

- 어려운 점은 시스템 수와 밸런싱입니다.
- 로스터, 계약, 전략, 훈련, 경기 결과, 시즌 캘린더, 장기 커리어가 모두 연결됩니다.
- 각 기능은 단순해 보여도 합치면 테스트해야 할 조합이 많습니다.

따라서 롤게임은 체스게임보다 초기 프로토타입은 빨리 보일 수 있지만, "오래 플레이해도 말이 되는 게임"으로 만드는 데 시간이 더 걸릴 수 있습니다.

## 6. 3주 운영 감각

3주를 21일로 보면:

```text
50시간 = 하루 약 2.4시간
80시간 = 하루 약 3.8시간
120시간 = 하루 약 5.7시간
170시간 = 하루 약 8.1시간
```

현실적인 운영:

- 하루 2~3시간이면 제출 가능한 MVP는 가능할 수 있습니다.
- 하루 3~5시간이면 LCK Split/국제전/장기 커리어까지 어느 정도 설득력 있게 만들 수 있습니다.
- 하루 6시간 이상을 꾸준히 써야 20시즌 밸런싱과 UI 폴리싱까지 욕심낼 수 있습니다.

## 7. 현재 판단

가장 현실적인 목표는 다음입니다.

```text
70~120시간
```

다만 제출만 생각하면 다음 범위도 가능합니다.

```text
50~80시간
```

안전한 전략:

- 1차 목표는 1시즌이 완전히 도는 게임으로 잡습니다.
- 2차 목표는 3시즌 연속 플레이로 잡습니다.
- 3차 목표로 20시즌 빠른 시뮬레이션 안정성을 확인합니다.
- 아시안게임, 커스텀 테마, 장기 이벤트는 핵심 루프가 돈 뒤 붙입니다.

핵심 리스크는 구현량보다 밸런싱입니다. "기능이 있다"보다 "선택이 결과에 영향을 준다"는 감각을 만드는 데 시간을 써야 합니다.
