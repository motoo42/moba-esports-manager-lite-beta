# LoL Esports 시즌과 대회 구조

## 1. 기본 방향

실제 LoL Esports와 LCK 캘린더를 기준으로 삼되, MVP에서는 각 이벤트를 게임에 맞게 압축합니다.

공식 LoL Esports 2025 핸드북 기준의 큰 흐름은 다음입니다.

```text
Split 1 -> First Stand -> Split 2 -> MSI -> Split 3 -> Worlds
```

이 프로젝트에서는 LCK 내부 시즌명을 더 실제 LCK 흐름에 가깝게 풀어서 사용합니다.

## 2. 게임 내 명칭 예시

| 구분 | 실제 명칭 | 게임 내 기본 명칭 | MVP 처리 |
| --- | --- | --- | --- |
| LCK 선발전 | LCK Cup | LCK Cup | 구현 |
| 국제전 1 | First Stand | First Stand | 구현 |
| LCK 전반기 | LCK Rounds 1-2 | LCK Rounds 1-2 | 구현 |
| 국제전 2 | MSI | MSI | 구현 |
| LCK 후반기 | LCK Rounds 3-5 | LCK Rounds 3-5 | 일반 시즌 구현 |
| LCK 후반기 축약 | LCK Rounds 3-4 | LCK Rounds 3-4 | 아시안게임 시즌 구현 |
| 국제전 3 | Worlds | Worlds | 구현 |
| 특별전 | 아시안게임 | Asian Games | 축약 구현 또는 3주차 후순위 |

| 추후 구현 | ENC | ENC | 향후 개선 |

## 3. 일반 연도 캘린더: 2025 시즌 참고

아시안게임이 없는 해의 기본 시즌입니다.

```text
LCK Cup
-> First Stand
-> LCK Rounds 1-2
-> MSI
-> LCK Rounds 3-5
-> Worlds
-> 오프시즌
```

설계 의도:

- 시즌 흐름이 단순합니다.
- 로스터 운영과 국제전 성과에 집중합니다.
- 장기 커리어에서 가장 자주 반복되는 기본 템플릿입니다.

## 4. 아시안게임 연도 캘린더: 2026 시즌 참고

아시안게임은 4년에 한 번 열리는 중요한 이벤트로 처리합니다.

```text
LCK Cup
-> First Stand
-> LCK Rounds 1-2
-> MSI
-> LCK Rounds 3-4
-> Asian Games
-> Worlds
-> 오프시즌
```

설계 의도:

- 아시안게임은 단순 보너스 이벤트가 아니라 한 해 일정과 선수 운영에 영향을 주는 이벤트입니다.
- 국가대표 차출, 피로도, 폼 변화, 사기 변화, 일정 압박 같은 이벤트를 만들 수 있습니다.
- 사용자는 클럽 성적과 국가대표 이벤트 사이에서 선수 관리 결정을 해야 합니다.

## 5. 아시안게임 4년 주기 규칙

커리어는 최대 20시즌까지 고려합니다.

아시안게임은 4시즌마다 한 번 발생합니다.

기본 규칙:

```text
seasonNumber % 4 === 0 -> 아시안게임 연도
그 외 -> 일반 연도
```

예시:

```text
1시즌: 일반 연도
2시즌: 일반 연도
3시즌: 일반 연도
4시즌: 아시안게임 연도
5시즌: 일반 연도
...
20시즌: 아시안게임 연도
```

나중에 실제 시작 연도를 받게 된다면 다음처럼 조정할 수 있습니다.

```text
isAsianGamesYear = (careerStartYear + seasonNumber - 1 - asianGamesBaseYear) % 4 === 0
```

MVP에서는 `seasonNumber % 4 === 0` 규칙으로 단순화합니다.

## 6. 로스터 규칙

- 사용자는 LCK 선수만 영입할 수 있습니다.
- LPL/LEC/LTA/LCP 등 타 리그 선수는 국제전 상대 팀에만 등장합니다.
- 타 리그 선수는 기본 MVP에서 영입 불가능합니다.
- 커스텀 테마를 사용해도 기본 게임 로직의 로스터 제한은 유지합니다.
- 타 리그 팀은 사전 정의된 데이터셋을 사용합니다.
- 현실성의 한계는 인정하되, MVP에서는 LPL/LEC/LTA/LCP 등 지역별 4개 팀 정도를 고정 데이터로 둡니다.

## 7. 진출 규칙 초안

- LCK Cup 결과로 First Stand LCK 대표를 결정합니다.
- LCK Rounds 1-2와 포스트시즌 결과로 MSI LCK 대표를 결정합니다.
- 일반 시즌에서는 LCK Rounds 3-5 최종 성적으로 Worlds LCK 대표를 결정합니다.
- 아시안게임 시즌에서는 LCK Rounds 3-4와 포스트시즌 흐름을 기준으로 Worlds LCK 대표를 결정합니다.
- 아시안게임은 시즌 성적과 별개로 국가대표 이벤트처럼 별도 로스터/선발 규칙을 둡니다.
- 아시안게임 로스터는 사용자의 팀 선수뿐 아니라 LCK 전체 선수 풀에서 선발될 수 있습니다.

## 8. LCK Cup 확정 포맷: 현실 기반 간소화

LCK Cup은 2026 현실 포맷을 기반으로 하되, 장기 커리어와 구현 속도를 위해 일부 요소를 간소화합니다.

### 그룹 배정

- 그룹은 고정하지 않습니다.
- 매 시즌 LCK Cup 시작 직전, 직전 LCK 시즌 최종 순위 기준으로 10팀을 정렬합니다.
- 1시즌은 직전 시즌 기록이 없으므로 팀 데이터의 초기 기준 순위를 사용합니다.
- 밸런스를 맞추기 위해 스네이크 분배를 사용합니다.
  - Baron: 1위, 4위, 5위, 8위, 9위
  - Elder: 2위, 3위, 6위, 7위, 10위
- 그룹명은 Baron/Elder를 유지하되, 매 시즌 구성은 바뀔 수 있습니다.

### 그룹 배틀

- 각 팀은 상대 그룹 5팀과 1번씩 경기합니다.
- 전체 25시리즈입니다.
- 5주 진행입니다.
- 한 주에 5시리즈가 진행됩니다.
- 사용자가 진행 버튼을 누르면 해당 주차의 5경기가 모두 처리됩니다.
- 1~4주차는 Bo3입니다.
- 5주차 Super Week는 Bo5입니다.
- 모든 경기는 Fearless Draft를 적용합니다.

### 그룹 포인트와 순위

- Bo3 승리: 해당 팀의 그룹에 1점.
- Bo5 승리: 해당 팀의 그룹에 2점.
- 팀 순위는 매치 승수, 세트 득실, 승률 기준으로 계산합니다.
- 그룹 포인트가 높은 쪽이 승자 그룹입니다.
- 그룹 포인트가 낮은 쪽이 패자 그룹입니다.
- 동률이면 그룹 전체 세트 득실로 판정합니다.
- 그래도 동률이면 상위 초기 시드가 있는 그룹을 우선 처리합니다.

### 그룹 배틀 종료 후 진출

- 승자 그룹 1~2위: 플레이오프 직행.
- 패자 그룹 1위: 플레이오프 직행.
- 승자 그룹 3~5위 + 패자 그룹 2~4위: 플레이-인 진출.
- 패자 그룹 5위: 탈락.

### 플레이-인

- 6팀 참가.
- 시드는 그룹 배틀 성적순으로 배정합니다.
- 싱글 엘리미네이션 방식으로 간소화합니다.
- 상위 2시드는 2라운드 직행입니다.
- 1라운드 Bo3:
  - 3시드 vs 6시드
  - 4시드 vs 5시드
- 2라운드 Bo3:
  - 1시드 vs 4/5 승자
  - 2시드 vs 3/6 승자
- 2라운드 승자 2팀이 플레이오프에 진출합니다.

### 플레이오프

- 5팀 참가.
  - 그룹 배틀 직행 3팀.
  - 플레이-인 통과 2팀.
- 전 경기 Bo5입니다.
- 싱글 엘리미네이션 방식으로 간소화합니다.
- 와일드카드:
  - 4시드 vs 5시드.
- 준결승:
  - 1시드 vs 와일드카드 승자.
  - 2시드 vs 3시드.
- 결승:
  - 준결승 승자끼리 경기.
- 결승 진출 2팀이 First Stand에 진출합니다.
- 결승 승자가 LCK Cup 챔피언입니다.

### 1차 구현에서 제외하는 현실 요소

- 상대 선택권 UI.
- 더블 엘리미네이션 브래킷.
- 복잡한 다자간 타이브레이커.
- 홈/어웨이와 첫 번째 선택권.
- 코치 보이스 효과.

## 9. First Stand / MSI 이벤트 구조

이 프로젝트에서는 First Stand와 MSI를 동일한 8팀 포맷으로 처리합니다.

- 참가 팀:
  - LCK 상위 2팀.
  - LPL 상위 2팀.
  - 3~6위 리그의 상위 1팀씩.
  - 총 8팀.
- 조별리그:
  - 4팀씩 2개 조.
  - 각 조 상위 2팀 진출.
- 토너먼트:
  - 진출 4팀이 4강전부터 싱글 엘리미네이션.
  - 최종 우승 팀 결정.

## 10. Worlds 이벤트 구조

이 프로젝트에서는 Worlds를 22팀 포맷으로 처리합니다.

- 참가 팀:
  - LCK 상위 4팀.
  - LPL 상위 4팀.
  - LCS 상위 4팀.
  - LEC 상위 4팀.
  - LCP 상위 2팀.
  - CBLOL 상위 2팀.
  - 총 22팀.
- 플레이인 스테이지:
  - 참가 팀 10팀.
  - LCK/LPL 4위 팀.
  - LCS/LEC 3, 4위 팀.
  - LCP/CBLOL 1, 2위 팀.
  - 상위 4팀만 조별리그 단계 진출.
- 조별리그:
  - 총 16팀.
  - 4팀씩 4개 조.
  - 각 조 상위 2팀 토너먼트 단계 진출.
- 토너먼트:
  - 8강 월드컵식 싱글 엘리미네이션.
  - 8강, 4강, 결승.

## 11. 아시안게임 이벤트 효과

아시안게임 연도에는 다음 효과를 줄 수 있습니다.

- 주전 선수가 국가대표로 차출될 수 있음.
- 차출 선수는 폼/명성/사기가 상승할 수 있음.
- 일정 부담으로 피로도가 상승할 수 있음.
- 금메달/메달 획득 시 선수 가치와 팀 명성이 상승할 수 있음.
- 대표팀 탈락이나 부진 시 멘탈/폼 이벤트가 발생할 수 있음.

MVP에서는 이 효과를 단순 이벤트 카드와 스탯 변화로 처리합니다.

## 12. 아직 정해야 할 세부 사항

- LCK Rounds 1-2, LCK Rounds 3-5, LCK Rounds 3-4의 정확한 주차/경기 데이터를 어디까지 재현할지.
- LCK 포스트시즌의 Bo3/Bo5, 더블 엘리미네이션, Road to MSI 브래킷을 게임 안에서 얼마나 압축할지.
- First Stand/MSI의 조별리그 경기 방식: 단판, Bo3, 더블 라운드 로빈 중 무엇으로 할지.
- Worlds 플레이인 10팀의 상위 4팀 선발 방식: 리그전, 스위스, GSL, 간단 점수식 중 무엇으로 할지.
- Worlds 조별리그 경기 방식: 단판, Bo3, 더블 라운드 로빈 중 무엇으로 할지.
- Worlds 토너먼트의 경기 단위: Bo5 전체 시뮬레이션 또는 압축 1경기 시뮬레이션.
