## Body

﻿## 목표
현재 메시지함 미디어 리뷰에만 사용 중인 AI API를, 게임 로직이 계산한 facts를 자연어로 해석하고 포장하는 보조 레이어로 확장한다.

AI는 게임 결과를 결정하지 않고, 이미 계산된 데이터를 더 읽기 좋은 리포트와 해설로 바꾸는 역할을 맡는다.

## 비목표
- AI가 승패, 선수 성장, 이적 성공률, 재정 결과를 직접 결정하지 않는다.
- AI가 제공되지 않은 선수명, 부상, 이적, 인터뷰, 징계 같은 사실을 새로 만들지 않는다.
- 모든 화면에서 상시 API를 호출하지 않는다.

## 핵심 원칙
- 기존 도메인 로직이 사실을 계산한다.
- AI는 facts를 받아 요약, 해설, 리포트, 프리뷰 문장만 생성한다.
- 실패 시 fallback 템플릿으로 게임 진행이 계속되어야 한다.
- API 키는 서버에만 두고 클라이언트에 노출하지 않는다.
- 비용을 통제하기 위해 설정, 캐싱, 호출 제한을 둔다.

## 선행 이슈 / 의존 관계
- 이미 구현됨: 메시지함 AI 뉴스 생성 파이프라인 v1
- 선행 권장: #34 매치엔진 화면
- 선행 권장: #71 개인 기록·통계 시스템
- 확장 연계: #69 경영 리포트, #70 코치진 리포트, #75 메타 패치 뉴스

## AI facts schema 공통 규칙
- facts는 화면/기능별로 명확히 분리한다.
- 모든 facts는 게임 내부에서 계산된 값만 사용한다.
- facts에는 다음 정보를 포함할 수 있다.
  - 날짜/대회/라운드
  - 팀/상대/스코어
  - 예상 승률/전력 차이
  - 선수 컨디션/폼/피로도
  - 주요 통계/평점
  - 스토브리그 제안/계약/시장 흐름
- AI 응답은 title/body/angle/safetyNotes 같은 검증 가능한 구조로 받는다.

## 활용 우선순위
1. 상대 리포트 AI 요약
   - 상대 전력, 최근 폼, 포지션별 강점, 우리 팀 약점 기반
2. 경기 프리뷰 / 경기 후 리뷰
   - 예상 승률, 핵심 선수, 승패 의미, 다음 경기 체크포인트
3. 시즌 결산 AI 리포트
   - 대회 성적, 시즌 흐름, 주요 경기, 아쉬운 점, 다음 시즌 과제
4. 스토브리그 구단 리포트
   - 포지션별 보강 필요성, 예산 여유, FA 시장 흐름
5. 선수단 리포트 문장화
   - 컨디션, 피로도, 폼, 선발/후보 상황
6. 선택형 게임 가이드 보강
   - 사용자가 요청할 때만 현재 화면/상태에 맞춘 짧은 설명 제공

## 저장 정책
- 뉴스/리포트처럼 기록으로 남길 텍스트는 career save에 저장한다.
- 현재 화면 도움말처럼 임시 성격이 강한 텍스트는 캐시만 사용한다.
- 같은 facts에 대한 재호출을 막기 위해 stable cache key를 둔다.

## 설정/비용 제어
- AI 기능 전체 ON/OFF
- 기능별 ON/OFF 또는 빈도 설정
- 개발자 모드에서 테스트 빈도 조절
- 실패 횟수 제한과 일정 시간 재시도 제한
- fallback 템플릿 필수

## 금지사항
- 제공되지 않은 현실 뉴스 생성 금지
- 실제 선수의 부상/징계/인터뷰 조작 금지
- 승패 또는 능력치 판정 조작 금지
- 비용 통제 없이 매 렌더마다 API 호출 금지

## 단계별 구현 계획
1. 기능별 facts schema 설계
2. 공통 AI generation service와 fallback interface 정리
3. 상대 리포트 AI 요약 프로토타입 구현
4. 경기 프리뷰/리뷰로 확장
5. 시즌 결산/스토브리그 리포트로 확장
6. 비용/캐시/설정 UI 강화
