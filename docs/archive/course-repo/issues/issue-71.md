﻿## 목표
선수별 경기 기록과 시즌/대회 단위 통계를 누적 관리하는 데이터베이스/통계 시스템을 도입한다.

이 시스템은 평점, MVP/베스트 수상, 계약 옵션, AI 리포트, 선수 커리어 기록의 핵심 기반이다.

## 비목표
- v1에서 실제 LoL 수준의 모든 세부 지표를 완벽히 구현하지 않는다.
- v1에서 모든 통계 화면을 완성하지 않는다.
- 과거 시즌 전체 기록을 실제 데이터처럼 정밀하게 백필하지 않는다.

## 선행 이슈 / 의존 관계
- 선행: #34 매치엔진 화면 또는 경기 detail payload 설계
- 선행 권장: #72 선수 능력치 시스템 구체화
- 후속: #74 리그·대회 베스트 및 MVP
- 후속: #73 선수 영입 옵션 조항
- 후속: #76 AI API 활용 확장

## 데이터 모델 변경
- 경기 단위 선수 stat schema
  - kills
  - deaths
  - assists
  - KDA
  - damage 또는 DPM
  - gold 또는 경제 지표
  - vision 또는 운영 기여도
  - objective participation
  - player rating
- 집계 단위
  - 경기별 기록
  - 대회별 기록
  - 시즌별 기록
  - 커리어 누적 기록
- 저장 위치
  - career save 내부 누적 데이터
  - 필요 시 `seasonState`와 선수 프로필 양쪽에서 접근 가능한 selector 제공

## 평점 공식 초안
- 경기 결과 보정
- KDA 기반 점수
- 포지션별 핵심 지표 가중치
- 상대 전력 보정
- 세트/대회 중요도 보정
- 과도한 폭주를 막기 위한 상한/하한 clamp

## UI/UX 변경
- 선수 프로필에 시즌/커리어 통계 표시
- 대회 현황 또는 시즌 결산에 주요 통계 리더보드 표시
- 경기 후 리뷰에서 주요 선수 기록 표시
- 향후 수상 시스템과 연결되는 평점 순위 표시

## 단계별 구현 계획
1. 경기 stat 타입과 저장 구조 정의
2. 매치 결과에서 stat을 생성하는 순수 함수 구현
3. 선수별/대회별/시즌별 집계 selector 구현
4. 평점 공식 v1 구현
5. 선수 프로필 또는 시즌 결산에 최소 UI 연결
6. 수상/계약 옵션/AI 리포트에서 활용 가능한 API 정리

## 밸런스 고려사항
- 승리 팀 선수만 과도하게 높은 평점을 받지 않도록 한다.
- 서포터/정글처럼 기록이 낮게 보일 수 있는 포지션도 공정하게 평가한다.
- 장기 시즌에서 누적 기록이 저장 용량을 과도하게 늘리지 않도록 한다.

## 테스트 계획
