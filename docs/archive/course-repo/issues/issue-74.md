﻿## 목표
리그 베스트, 대회 베스트, 대회 MVP를 선정하는 수상 시스템을 도입한다.

#71 개인 기록·통계·평점 데이터를 기반으로 선수 성과를 시즌 결산, 선수 커리어, 메시지/뉴스, 계약 옵션에 연결한다.

## 비목표
- 실제 LCK/국제대회 수상 규칙을 완벽히 재현하지 않는다.
- v1에서 팬 투표, 미디어 투표, 선수단 투표까지 세분화하지 않는다.
- 통계 시스템 없이 임의 선정만으로 수상 시스템을 완성하지 않는다.

## 선행 이슈 / 의존 관계
- 선행: #71 개인 기록·데이터베이스·통계 시스템
- 연계: #73 선수 영입 옵션 조항 추가
- 연계: #76 AI API 활용 확장 계획

## 수상 종류
- 대회 MVP
- Finals MVP
- 포지션별 대회 베스트
- LCK 리그 베스트
- 시즌 베스트 또는 올해의 선수 후보

## 선정 기준
- 선수 평점 평균
- 최소 경기 수 또는 최소 세트 수
- 팀 성적 보정
- 결승/플레이오프 등 중요 경기 보정
- 포지션별 핵심 지표 보정
- 동률 처리 기준

## UI/UX 변경
- 시즌 결산 메뉴에 주요 수상자 표시
- 대회 현황 완료 화면에 대회 MVP/베스트 표시
- 선수 프로필 커리어에 수상 기록 누적
- 메시지함 또는 뉴스에서 주요 수상 결과 알림

## 데이터 모델 변경
- award type
- award season/year
- competition id
- player id/team id
- 선정 근거 요약
- 선수 커리어 표시용 award history

## 단계별 구현 계획
1. 수상 타입과 데이터 모델 정의
2. 대회 종료 시점에 후보군과 최소 경기 수 필터링
3. 평점/통계 기반 선정 로직 구현
4. 시즌 결산/대회 완료 화면에 표시
5. 선수 프로필 수상 기록 연동
6. 계약 옵션/AI 뉴스와 연결 가능한 이벤트 payload 정리

## 밸런스 고려사항
- 우승팀 선수가 유리하되, 항상 우승팀에서만 나오지는 않도록 한다.
- 포지션별 지표 차이를 반영해 서포터/정글이 불리하지 않게 한다.
- 적은 경기만 뛰고 고평점을 받은 선수가 과도하게 선정되지 않도록 최소 경기 수를 둔다.

## 테스트 계획
- 평점 순위 기반 수상자 선정 테스트
- 최소 경기 수 미달 제외 테스트
- 동률 처리 테스트
- 시즌 결산/선수 프로필 표시 테스트
- 계약 옵션 연동용 이벤트 생성 테스트

- [ ] 대회 MVP/포지션별 베스트 선정 로직 구현
