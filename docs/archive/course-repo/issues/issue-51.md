## 배경

현재 메시지함 메뉴는 안 읽은 메시지가 있을 때 좌측 메뉴에 개수를 표시합니다. 그런데 이 숫자가 메시지함 로고 하단에 배치되어 메뉴 항목의 높이가 늘어나는 문제가 있습니다. 좌측 메뉴는 항목 높이와 정렬이 일정해야 하므로, 안 읽은 메시지 개수는 텍스트 옆 배지 형태로 표시하는 편이 적절합니다.

## 요구사항

- 안 읽은 메시지 개수는 `[메시지함]` 텍스트 오른쪽에 배지 형태로 표시합니다.
- 배지는 메뉴 항목의 높이를 늘리지 않아야 합니다.
- 안 읽은 메시지가 1~99개인 경우 실제 개수를 표시합니다.
- 안 읽은 메시지가 100개 이상인 경우 `99+`로 표시합니다.
- 안 읽은 메시지가 0개인 경우 배지를 표시하지 않습니다.
- 기존 메시지함 메뉴 클릭/이동 동작은 유지합니다.

## 완료 기준

- 안 읽은 메시지가 있어도 메시지함 메뉴가 위아래로 길어지지 않습니다.
- 메시지 개수 배지가 `[메시지함]` 텍스트 오른쪽에 안정적으로 정렬됩니다.
- 100개 이상의 안 읽은 메시지는 `99+`로 표시됩니다.
- 안 읽은 메시지가 없을 때는 배지가 보이지 않습니다.
- 좌측 메뉴의 다른 항목들과 높이/정렬이 일관됩니다.

## 범위 제외

- 메시지 읽음/안 읽음 처리 로직 변경은 이번 이슈 범위가 아닙니다.
- 메시지함 목록 화면 자체의 UI 개편은 별도 이슈에서 다룹니다.
