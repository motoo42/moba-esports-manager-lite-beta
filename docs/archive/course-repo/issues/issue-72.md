## Body

﻿## 목표
FM/FIFA식 세부 능력치 체계를 설계하고, 기존 `overall`/별점 중심 선수 평가를 포지션별 세부 능력치 기반으로 확장한다.

이 시스템은 매치엔진, 선수 성장, 코치진, 통계, AI 리포트, 밸런스 패치의 기반이 된다.

## 비목표
- 이번 이슈에서 모든 선수 능력치를 완벽하게 밸런싱하지 않는다.
- 실제 경기 데이터를 기반으로 한 정밀 스카우팅 모델까지 만들지는 않는다.
- 챔피언별 숙련도나 메타 적응력은 별도 후속 이슈에서 다룬다.

## 선행 이슈 / 의존 관계
- 선행: 현재 선수 데이터 구조 파악 및 마이그레이션 계획 수립
- 병행 가능: #34 매치엔진 화면 구현
- 후속: #71 개인 기록·통계, #70 코치진, #75 밸런스 패치

## 데이터 모델 변경
- 선수 기본 능력치 카테고리 예시
  - 라인전
  - 교전
  - 운영
  - 챔피언 폭
  - 안정성
  - 클러치
  - 성장성
  - 멘탈/압박 대응
- 포지션별 가중치 정의
  - TOP: 라인전, 안정성, 교전, 사이드 운영
  - JUNGLE: 운영, 초반 설계, 교전, 오브젝트 판단
  - MID: 라인전, 교전, 클러치, 챔피언 폭
  - BOT: 교전, 안정성, 후반 캐리력, 라인전
  - SUPPORT: 운영, 시야/이니시, 교전 설계, 안정성
- `overall`은 세부 능력치 기반으로 계산하거나, 과도기에는 기존 값을 유지하며 보정값으로 사용한다.

## 등급 기준
- 50 미만: 약점이 뚜렷함
- 50-59: 보통 이하
- 60-69: 리그 평균권
- 70-79: 좋은 선수
- 80-89: 리그 상위권
- 90 이상: 월드클래스

## UI/UX 변경
- 선수 상세 모달 또는 프로필에 세부 능력치 표시
- 로스터 관리 화면에서는 핵심 능력치만 요약 표시
- 별점/오버롤은 유지하되, 세부 능력치를 더 깊은 판단 근거로 제공

## 단계별 구현 계획
1. 능력치 타입과 기본 스키마 정의
2. 포지션별 가중치와 오버롤 계산식 초안 구현
3. 기존 선수 데이터에 임시 능력치 자동 생성 또는 수동 부여
4. 선수 상세 UI에 세부 능력치 표시
5. 매치엔진과 연결 가능한 계산 함수 분리
6. 밸런스 검증용 테스트와 샘플 데이터 점검

## 밸런스 고려사항
- 이름값이 높은 선수에게만 능력치가 과도하게 몰리지 않도록 한다.
- 유망주는 현재 능력과 성장성을 분리한다.
- 포지션별 오버롤 산식이 서로 다른 역할을 공정하게 반영해야 한다.
- 기존 게임 난이도가 급격히 변하지 않도록 기존 `overall`과 비교 검증한다.

## 테스트 계획
- 포지션별 오버롤 계산 단위 테스트
- 기존 선수 데이터 마이그레이션 테스트
- 선수 상세 UI 렌더 테스트
- 매치 시뮬레이션 입력값 호환성 테스트
