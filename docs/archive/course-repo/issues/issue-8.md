## 목표

프로젝트용 GitHub 레포에서 README와 문서가 현재 구현 상태를 정확히 보여주도록 정리한다.

README는 공개될 수 있으므로 베타 링크, MongoDB URI, 비밀번호 같은 민감하거나 제한 공개 중인 정보가 노출되지 않아야 한다.

## 작업 내용

- README 최신화 상태 점검
- 공개 README에서 실제 베타 링크 직접 노출 제거
- 베타 링크는 별도 안내 또는 개인 전달 방식으로 관리
- 다음 문서 간 역할 정리
  - `README.md`: 공개 프로젝트 소개
  - `CODEX_HANDOFF.md`: Codex/에이전트 이어받기
  - `docs/development-checklist.md`: 공식 체크리스트
  - `IMPLEMENTATION_ORDER.md`: 현재 스프린트 안내판
  - `docs/beta-tester-guide.md`: 베타테스터 안내
  - `docs/beta-deploy-guide.md`: 배포 절차
- 문서에 오래된 구현 상태나 중복 체크리스트가 남아 있는지 확인

## 완료 기준

- README가 현재 구현 상태와 맞음
- 공개 문서에 실제 베타 링크나 secret이 없음
- 문서별 역할이 겹치지 않음
- 신규 개발자가 README와 핸드오프 문서만 보고 현재 상태를 이해할 수 있음
