## Body

## 목표
라이브 매치 **재생(연출) 레이어**를 더 사실적으로 정교화한다.

#34에서 구현한 매치엔진 v1은 이미 결정된 경기 결과를 밴픽/문자중계/스탯 패널/세트 결과로 보여주는 구조다. #87은 이 구조를 계승하되, 다음 두 축을 후속으로 강화한다.

- **② 연출 편향**: 골드, 레벨, KDA, 솔로킬, 오브젝트, 경기 페이스를 선수 능력치·라인 우열·아이템 스파이크에 따라 더 자연스럽게 편향한다.
- **③ 기세(승률) 그래프**: 시점별 스냅샷 델타를 바탕으로 승률 곡선과 주요 사건 마커를 보여준다.

## 핵심 전제
- 경기 결과, 세트 승패, 스코어는 기존 `match-simulation`이 source of truth다.
- #87은 **이미 결정된 결과를 재생하는 live-match 연출 레이어만 정교화**한다.
- 결과 계산은 건드리지 않는다.
- 작동 중인 3시즌 시뮬레이션을 흔들지 않는다.
- 모든 편향은 기존 불변식 안에서만 작동한다.
  - 블루킬 = 레드데스
  - 레드킬 = 블루데스
  - 승자가 킬 우세
  - 마지막 넥서스는 실제 승자
  - 부활 가드 재사망 0
  - 동일 seed → 동일 결과

## 스코프
### ② 연출 편향
선수 능력치, 라인 우열, 아이템 스파이크가 생긴 뒤 live-match의 사건과 스탯 분배를 더 자연스럽게 만든다.

- `matchTimeline` 조정
  - 솔로킬/교전 발생 측과 라인을 선수 능력치·라인 우열로 편향
  - 게임 길이와 킬 페이스를 dominance뿐 아니라 팀 스케일링/드래프트 성향으로 보정
  - 오브젝트 획득 가중을 챔피언 성향, 시야/이니시, 아이템 파워스파이크로 보정
- `matchStats` 조정
  - 골드/레벨/KDA 분배를 역할 고정 상수에서 선수 능력치·라인 우열 기반으로 개선
  - #86의 시점별 아이템 산출이 준비되면 아이템 슬롯을 실제 시간축에 맞춰 교체

단, 이 작업은 #72와 #86의 입력값이 있어야 제대로 연결할 수 있다.

### ③ 기세(승률) 그래프
시점별 경기 흐름을 한눈에 볼 수 있도록 live-match 화면에 기세 그래프를 추가한다.

- `buildMomentumSeries` 순수 함수 생성
  - 입력: timeline + `getMatchSnapshotAt` 기반 시점별 스냅샷
  - 출력: 시점별 블루 우세 승률 0~1
  - 끝점은 실제 승자에 앵커링
  - 동일 seed와 동일 timeline이면 동일 곡선 생성
- 승률 곡선 렌더
- visible 주요 사건을 시간축 마커로 부착
  - 사건명
  - tone: blue / red / neutral
- 현재 재생 시점 표시
- `LiveMatchScreen` 내부에 작지 않은 1/n 영역으로 배치
- 그래프 시각 레퍼런스는 사용자가 별도로 제공 예정

## ③ 기세 그래프 — 시각 레퍼런스 & 사양 (사용자 제공, #87 작업 시 참고)

> 작성: Claude. 레퍼런스 **이미지는 사용자가 본 이슈에 첨부 예정**. 결과계산 불변·연출 레이어 한정 전제는 본문 그대로 계승.

### 레이아웃 (위 → 아래)
1. **상단바**: (좌) 팀 로고+이름+승률%(파랑) / (중앙) `N세트` 칩 / (우) 승률%+이름+로고(빨강). 이름 아래 **승률 비율 막대**(파랑|빨강, 예 `76|24`).
2. **현재 사건 배너**(차트 우상단에 떠 있는 칩): `[아이콘] MM:SS · 팀명 · 사건명` (예: `41:42 한진 브리온 · 넥서스 파괴`). 재생 현재 시점의 최신 사건으로 갱신.
3. **사건 점 행**(차트 바로 위 스트립): 사건 시점마다 tone색(파랑/빨강) 작은 점, 일부 오브젝트 아이콘. 현재 시점은 큰 점.
4. **기세 area 차트**:
   - X = 경기 시간(분), 0~경기길이 / Y = 승률(위 블루 100% · 중앙 50% · 아래 레드 100%)
   - 블루 우세 → 중앙에서 **위로 파랑 영역**, 레드 우세 → **아래로 빨강 영역**. 노이즈 있는 곡선.
   - 끝점 = **실제 승자 100% 근처**(넥서스 스파이크). 현재 재생 시점 = **세로선**.

### 사건 스티커 (사용자 추가 요구)
- **에픽 몬스터(드래곤/영혼/장로/바론/전령)·주요 오브젝트·주요 교전** → 곡선 위 해당 시점에 **작은 스티커(태그/이모티콘/아이콘)** 부착해 "언제 무슨 일"이 한눈에. 기존 `LiveDragonIcon`/`LiveMatchIcon` 재사용 권장.

### 선택 (옵션 — v1 제외 가능)
- 마커 **호버 툴팁**(시점 사건 표시): 구현해도/안 해도 됨.

### #87 v1 매핑
곡선 + 끝점 앵커 + 사건 마커 + 현재 재생 시점 + 1/n 영역 = **기존 v1 그대로**. "스티커"는 사건 마커의 구체화. 호버 툴팁은 기존 v1 제외 항목과 일치(옵션).
