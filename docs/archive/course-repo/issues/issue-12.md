## Body

## Summary

현재 커리어 저장/불러오기 기능은 있지만, 저장 슬롯을 삭제하는 기능이 없다. 베타테스트와 장기 플레이 과정에서 불필요한 저장 데이터가 계속 쌓일 수 있으므로 `데이터 저장` 화면에 저장 삭제 기능을 추가한다.

## Why

- 잘못 만든 커리어나 테스트 저장을 정리할 수 있어야 한다.
- 베타테스터가 여러 번 새 커리어를 만들 때 저장 목록이 지저분해지는 문제를 줄인다.
- MongoDB `careerSaves` 컬렉션에 불필요한 문서가 계속 남는 것을 방지한다.

## Scope

- `/saves` 데이터 저장 화면에 저장 삭제 버튼 추가
- 삭제 전 확인 모달 또는 확인 프롬프트 표시
- 삭제 성공 후 저장 목록 새로고침
- 현재 로드 중인 저장 슬롯을 삭제할 때의 UX 정의
  - 추천: 삭제는 가능하되, 삭제 후 현재 커리어는 메모리에서 계속 진행 가능
  - 다음 수동 저장 시 새 저장으로 저장하도록 안내
- 삭제 실패 시 에러 메시지 표시
- 커리어 시작 전 저장/불러오기 패널에서도 삭제 가능 여부 검토

## Technical Notes

- 서버에는 이미 `DELETE /api/saves/:saveId` 계열 API가 있는지 먼저 확인한다.
- API가 있다면 프론트 `SaveManager`에 연결한다.
- API가 없다면 Controller / Service / Repository 구조에 맞춰 추가한다.
- revision conflict 정책과 충돌하지 않도록 삭제 응답/에러 처리를 정리한다.

## Test Plan

- 저장 목록에서 저장 슬롯 삭제 버튼이 표시된다.
- 삭제 확인에서 취소하면 아무 변화가 없다.
- 삭제 확인에서 확정하면 API 호출 후 목록에서 제거된다.
- 삭제 실패 시 사용자에게 실패 메시지가 표시된다.
- 현재 활성 저장을 삭제했을 때 자동 저장/수동 저장 상태가 깨지지 않는다.
- `npm.cmd test`
- `npm.cmd run build`

## Priority

베타테스트 전 편의성 개선 항목. 메시지함/뉴스 1차 구현과 비슷한 우선순위로 관리하되, 작업 범위는 비교적 작을 가능성이 높다.
