# UI 리디자인 작업 폴더

전체 UI를 갈아엎되 **메뉴/네비게이션/바로가기/버튼 동작/라우팅 등 뼈대는 그대로 유지**하는 작업을 위한 문서 모음.

## 접근 방식 (확정)

- **방법 1 — 순수 CSS 리스킨**: 클래스명 계약을 유지하고 `src/shared/styles/global.css`의 **토큰값만 교체**해 전 화면을 일괄 리스킨. 마크업(요소 배치) 변경은 규약 확정 후 필요한 화면만 별도로(방법 2).
- 전제: 색/간격/타이포 등은 **토큰(CSS 변수)** 한 곳에서 관리 → 토큰만 바꾸면 화면이 따라온다.
- 현재 문제: 색 값의 약 78%가 토큰을 우회한 **하드코딩**이라, 토큰 교체만으론 일부만 반영됨 → **토큰화 패스**가 선행돼야 함.
